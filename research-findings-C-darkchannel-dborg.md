## TL;DR
- Both executions followed their frozen packages near-verbatim (waves, branches, read-order, STATUS boards) — the freeze mechanism works.
- Biggest dark-channel failure: 62/62 green ≠ acceptable output; the missing visual-quality NFR cost an entire re-plan+rebuild iteration (v2 quality editor).
- `[REAL]`-typed contract cases are the single highest-value plan artifact: they caught 7 integration defects in it5 and 2 more in it7 that unit-green subagent work hid.
- Stale anchors rot fast: file moves, archived epics, and a hardcoded "frozen v2.0" broke the it7 baseline within days; the manual "re-ground anchors" task should be automatic.
- Env facts living in the user's head (Node 22, SunBrowser-holds-HeyGen-login, key set/unset) surfaced mid-build; plans need a machine-checkable env manifest.
- db-organization's late discoveries were research-accuracy errors (stream-C wrong on install-state), arguing for double-sourcing any claim that gates destructive actions.
- Done-claims were policed well (E4-1 reopened at 9/12; T-E2.1-08 held, not fake-greened) — keep coordinator-verifies-contract as law.
- Handoffs (KICKOFF + inlined launch prompt + memory + /next-session-prompt) loaded first-try in both execs; the one hazard was plan artifacts left uncommitted at plan-session end.
- Fernando re-types the same run-policy (model tiering, autonomy consent, quiesce-other-sessions) every session — plan-it should freeze it into the package.
- Only ~13% of the 184 db-refactor cases persist as regression cover; persistence class should be planned per case, not improvised after the build.
