## TL;DR
- Smart-composer handoff was frictionless (KICKOFF.md + frozen PRD + HTML mockup + /next-session-prompt); exec never asked where anything was.
- Biggest smart-composer miss: no measured latency baseline → unplanned Redis search-cache workstream ("5s @ search") shipped outside epics E0–E6.
- Automated Stop hooks, not Fernando, were the main mid-run "interventions" — plans don't encode workspace bookkeeping obligations (.taskstate files).
- Test Contracts worked as real DoD in both pairs: 20-case (5 [REAL], anti-fake-green rule) and 21→26/26; "done" never accepted without them.
- gate-check caught a genuine planning defect ("Count: 20" vs 21 rows) — but only at handoff; move the lint to write-time.
- Plan committed to an unsupported mechanism (Task-tool PreToolUse matchers) → descope; add a harness-capability spike to discovery.
- Model tiering decided at plan time (sonnet for spec'd epics, fable for the dangerous E5 twin-wiring slice) transferred cleanly and worked.
- Fan-out agents repeatedly finish without delivering reports; mandate write-to-disk + content-bearing final message.
- Roadmap session: implementation docs lag reality (~85%-done items still "planned"); v3 needs a status-reconciliation pre-phase and an analysis-only mode.
- Keep: _RESIDUE.md closeout notes, fail-open guard hooks, batched decision gates, auto-sizing that skipped squad ceremony for a 1,177-line repo.
