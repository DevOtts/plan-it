## 2026-07-08 — v3.1.0 enhancement wave (enforcement reach; from the v2-vs-v3 field trial)
Source: a head-to-head trial (`.fable-it-reports/v2-vs-v3-comparison-2026-07-08.md`) ran v2.1.0 and v3.0.0 on the **same external brief** (a credential-vault CLI — an *external* project, not plan-it's own dogfood). All three are enforcement/checker concerns, not statechart or CONTRACT changes; the protected v2 core stays byte-identical and `machine-diff` still reports additive-only. Fixes made on branch `v3.1/enforcement-reach` and mirrored.

- **W3.1-2 (fixes E1) — freeze hardening decoupled from `--dir` and from plan-it's own path.** Before: `casesReviewed` (C-W1-03) and the RUN-POLICY-in-package requirement fired only in `freeze --dir` mode, and `--dir` hardcoded `delivery/v3/CONTRACT.md` (the dogfood path). A real project freezes `delivery/CONTRACT.md` **positionally**, so both checks were skipped and a package with no RUN-POLICY froze at exit 0 (exactly what the v3 arm shipped). After: a new `resolveRunRoot(contractPath)` helper deems a freeze "v3-backed" when the contract sits at a run root's canonical delivery path (`<root>/delivery/CONTRACT.md` or `<root>/delivery/v3/CONTRACT.md`) **and** that root carries `.plan-it/state.json`. The v3 checks key off that run root, not the flag. `--dir` now also finds `delivery/CONTRACT.md` (real-project path preferred, dogfood path as fallback). Deliberately narrow: a freestanding v2 contract (no delivery/ run root, no sibling state) resolves nothing → **byte-identical v2 behavior** (proven by `T-W31-2d` and the untouched `T-E3-04/05/06`). New fixtures: `real-project-nopolicy/` (positional + `--dir` both now fail on RUN-POLICY → `T-W31-2a/2b`), `real-project-good/` (valid contract + RUN-POLICY + `casesReviewed:true` → freezes clean, proves no over-block → `T-W31-2c`).
- **W3.1-1 (fixes E2) — `stripCode` tolerates a line-wrapped inline code span.** Before: `` /`[^`\n]*`/ `` refused to strip a backtick span that wrapped across a newline, so a `<id>` token inside it tripped the placeholder scan (the v3 arm had to reword its CONTRACT to appease the checker). After: `` /`[^`]*?`/ `` (lazy, newline-tolerant) with a guard that KEEPS any span straddling a blank line (a paragraph break — never a real inline span), so an unterminated backtick can't swallow the document and a **bare-prose** `<id>` still fails closed.
