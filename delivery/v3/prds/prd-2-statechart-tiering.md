## 1. Problem & goals

The v2.1 protected core (`machine.json`, `scripts/gate-check.mjs`, `scripts/hooks/planit-guard.mjs`) is a strictly linear 15-state statechart with no environment-reality gate and no model-tiering enforcement. The 15-pair field study (`research-SYNTHESIS.md` #3, #8) found two concrete failure classes this lane must close:

1. **No runtime reality probe.** Plans were authored against the repo, not the live/tooling environment; SKILL.md Phase 3 already prose-instructs live-grounding (`SKILL.md:287-317`) but nothing enforces it, times it out safely, or scales it by run size. CONTRACT.md W2 (`C-W2-01..04`) makes this a machine-legible, guarded gate.
2. **No model-tiering enforcement.** `research-SYNTHESIS.md` #8 (RUN-POLICY) is already text in `CONTRACT.md:52-65`, but nothing (a) requires every epic to carry a parseable Tier Table, (b) blocks a hardcoded model ID from ever landing in a plan artifact, or (c) verifies the CONTRACT-level RUN-POLICY block itself is present and seeded from the run's own G1 decisions before `freeze` succeeds. CONTRACT.md W3 (`C-W3-01..03`) + TEST-CONTRACT-REVIEW area G (`G1-G2`) close this.

Goals (bound to the 14 IDs this squad owns):
- Enforcement: `C-W2-01` `C-W2-02` `C-W2-03` `C-W2-04` `C-W3-01` `C-W3-02` `C-W3-03` (7 rows, CONTRACT.md:18-24).
- Draft/approved: `E1` `E2` `E3` `E4` `E5` `G1` `G2` (7 areas, TEST-CONTRACT-REVIEW.md:32-46).
- Hard constraint carried into every design below: protected v2 core untouched; every statechart/verb change is additive; v2 runs need zero `state.json` migration (E3); no hardcoded model IDs anywhere, including this PRD and its epics file (tier words only).

## 2. Known tension — E1 (byte-pin) vs C-W2-01 (new `preflight` state)

**The conflict.** `delivery/TEST-CONTRACT-REVIEW.md:33` (E1, approved-as-is): *"machine.json byte-identical to v2 fc6abc8 copy (core untouched) — regression pin."* `delivery/v3/CONTRACT.md:18` (C-W2-01, frozen): *"`machine.json` contains `preflight` state with edges discovery→preflight→contract."* Verified today: `git show fc6abc8:machine.json` diffs clean against the live `machine.json` (zero bytes changed) — E1 currently holds trivially, precisely because no `preflight` state exists yet. Adding one to satisfy C-W2-01 literally breaks E1 literally. This also contradicts `delivery/design/v3-architecture.md:26-28`, which explicitly ruled "NO new machine.json state" for the reality-probe concept and routed it through a `gate-check testconv` subcommand instead — i.e. the frozen CONTRACT.md (authored after that architecture decision, post-G2) reopened a question the architecture doc had already closed. Both E1 and C-W2-01 are squad-immutable text (guardrail 4: "gaps surface as PROPOSED-AMENDMENT … never as squad-side contract edits").

**D1 — Two options, recommendation: Option 2.**

- **Option 1 — preflight in a sibling/overlay file.** Keep root `machine.json` permanently byte-identical to the `fc6abc8` copy (E1 passes unmodified, zero amendment to E1 needed). Ship `preflight` in a new file (e.g. `machine-v3.json`) selected by the same Phase-0 `machine` selector state.json already needs for `machine-replan.json` (E3, D6 below). *Cost:* `C-W2-01` as literally worded — "`machine.json` contains `preflight`" — would then be false of the literal root file; `tests/v3/machine-shape.mjs` would have to point at a different filename than the one the case names, which is itself a wording break on an enforcement row, not just a draft case. It also doesn't fit the architecture's own rationale for the sibling-file pattern (§3: siblings exist for *alternate initial transitions* that break linearity — `preflight` is a plain linear insertion, not a branch, so it doesn't need a branch-shaped fix).

- **Option 2 (recommended) — stored v2 regression copy + additive-only structural diff on the live machine.** Freeze a byte-pinned copy of the pre-v3 graph at `tests/fixtures/v2/machine.v2.fc6abc8.json` (== `git show fc6abc8:machine.json`, verified identical to the current live file today). Insert `preflight` directly into the live `machine.json` as a genuinely additive state (new node + two new edges; every v2 state, edge, event name, and guard name is preserved byte-for-byte — nothing renamed or removed). Add a new **additive-only structural diff guard**, `gate-check machine-diff <live> <baseline>`, that walks both graphs and fails if the live graph is missing any v2 state, drops any v2 transition, or changes any existing guard mapping — it only *permits* pure additions. E1's protection goal ("core untouched") is preserved at the semantic level the guardrail actually cares about (no v2 behavior lost), while `C-W2-01` is satisfied literally against the real, single, canonical `machine.json` file every other tool (`gate-check state`, SKILL.md Phase 0, `tests/run-contract.mjs` T-E1-*) already resolves by default.

  **PROPOSED-AMENDMENT-1** (for the coordinator, not applied by this squad): reword E1 in the next dated `decisions.md` / CONTRACT.md changelog entry from *"machine.json byte-identical to v2 fc6abc8 copy"* to *"live machine.json is a structural superset of the byte-pinned v2 regression copy at `tests/fixtures/v2/machine.v2.fc6abc8.json` — no v2 state, transition, event name, or guard mapping removed or renamed; verified by `gate-check machine-diff`."* Until the coordinator applies this, `T-B1-02` below runs against Option 2's actual mechanism and documents the wording gap inline rather than silently reinterpreting frozen text.

**Interpretation flag on "contract" in C-W2-01's edge chain.** No v2 state is named `contract`; the nearest analog is `backboneFreeze` (where `CONTRACT.md` is authored, `machine.json:150-160`) at the far end of a chain `discovery → preflight → synthesis → specAuthoring → decisionGate → coherencePass → freezeGate → backboneFreeze`. This squad reads "contract" in the case prose as shorthand for *"the contract-authoring track downstream of discovery,"* not a literal third node, and implements `preflight` as a single additive insertion `discovery → preflight → synthesis` (D2), with `machine-shape.mjs` asserting reachability from `preflight` to `backboneFreeze` rather than a literal edge to a node named `contract`. Flagged here so the coordinator can confirm or correct the reading; it does not require a CONTRACT wording change if confirmed, only a documented interpretation.

## 3. Solution design (D1 continued → D13)

**D2 — `preflight` state mechanics.** `machine.json:82-93` (`discovery`/`synthesis` nodes). Insert:
```json
"preflight": {
  "meta": { "phase": 4, "title": "Preflight (env facts)",
    "does": "Probe the real environment (tools, versions, live reachability) at S/M/L tiering; timeout-safe; emits ENV-FACTS.md." },
  "on": { "PREFLIGHT_VERIFIED": { "target": "synthesis" } }
}
```
and change `discovery`'s `FANOUT_COMPLETE` target from `synthesis` to `preflight` (`machine.json:90`). No existing state, event, or guard name changes — additive per D1 Option 2.

**D3 — ENV-FACTS.md tiering (C-W2-02).** SKILL.md's Phase-3 live-grounding gate already enumerates exactly six checks (`SKILL.md:290-317`): config reachability, live registry/canonical identifiers, deployed-vs-installed-vs-in-use, credential validity, dependency sets from actual usage, and the "code change vs external connection" split. These six become the **S-shape minimal probe set**, formalized as a new `references/formats.md` §9 ENV-FACTS schema (probe id, check, status ∈ {PRESENT, ABSENT, TIMEOUT}, evidence). **M/L full set** adds: (7) tool-availability probe per `run:` command referenced anywhere in the plan's draft Test Contract (feeds C-W2-03), (8) caller/consumer-surface map, (9) full-tree secret-scan-on-import — the two corollaries already named at `SKILL.md:314-317` plus the new tool-availability probe. Size (S/M/L) is already recorded at G1 in `.plan-it/state.json:9` (`"shape": 3` / precedent field `size`) — tiering reads that existing field, no new state.json key required.

**D4 — probe execution + 10s timeout (C-W2-04).** New `gate-check preflight <S|M|L> [--dir <target>]` subcommand (`gate-check.mjs`, alongside `cmdVerify`/`cmdFreeze` at `gate-check.mjs:32-86`). Each probe runs as a child process via `execFileSync(..., { timeout: 10_000 })`; a timeout throws `ETIMEDOUT`, caught per-probe (never aborts the whole run), recorded as `TIMEOUT` in ENV-FACTS.md, and the loop continues to the next probe — mirrors the existing fail-soft pattern already used by `planit-guard.mjs`'s outer `try{} catch{ allow(); }` (`scripts/hooks/planit-guard.mjs:40-65`) but fails a probe closed (recorded, not silently dropped) rather than allowing.

**D5 — ABSENT-tool → `contract` verb failure (C-W2-03).** This squad owns the ENV-FACTS *schema and probe semantics* (D3/D4); the `contract` verb itself is Squad A's W1 lane (`00-program-plan.md:9`, `C-W1-01`). Cross-squad contract: any Test Contract `run:` cell whose leading executable matches an ENV-FACTS probe recorded `ABSENT` is a fail condition for `gate-check contract`. This PRD specifies the schema field (`status: "ABSENT"`) Squad A's verb reads; `T-B1-05` below exercises it end-to-end against a shared fixture and will need Squad A's `contract` verb to exist — flagged as a Wave-1 cross-squad dependency, not a blocker to this PRD's own design.

**D6 — Phase-0 machine selector + `machine-replan.json` (E2, E3).** New optional `.plan-it/state.json` key `machine` (absent in every existing v2 run — additive, matches E3's "v2 runs unaffected" requirement literally: `findMachine()` in `gate-check.mjs:201-208` already defaults to `machine.json` when no explicit path is passed, so E3 is satisfied by *not adding a required field*, only documenting the optional one). `machine-replan.json` is authored as a new sibling file validating against the identical JSON-shape gate-check.mjs already enforces for `machine.json` (`tests/run-contract.mjs:43-102`, T-E1-01..05 pattern reused, not duplicated as a new subcommand): parses as JSON, `initial` ≠ `"intake"` (collision guard), every transition target resolves, exactly one `type: "final"` state named `done`.

**D7 — freezeGate known-gaps ledger (E4, EC-B7).** `research-SYNTHESIS.md:100-102` names the exact incident: *"the plan's own handoff lint flagged 'INV-2 … has no regression test' yet the package froze anyway."* `machine.json:137-149` (`freezeGate`, guard `gateRecorded`) currently only requires owner+date (`gate-check.mjs:236-245`, `cmdState`'s per-gate loop). Extend that same loop: when `state.state` has passed `freezeGate` (gate `G3`), additionally require `state.gates.G3.knownGaps` to be an array where every row's `disposition` ∈ {`fix`, `waive`, `case-ify`} — any row missing a disposition fails `G3_APPROVED` with the row named. Additive: only fires when a `knownGaps` array is present, so pre-v3 state files with no such key are unaffected (E3-style backward compatibility).

**D8 — decisionGate credential checklist (E5, backlog #10).** Same extension point as D7, applied to `G2`: `state.gates.G2.credentials` rows, each `status` ∈ {`procured`, `GATED-with-owner`}; a row with `status: "unprocured"` and no `owner` field fails `G2_ANSWERED`. On accept, `GATED-with-owner` rows are additionally required to appear verbatim in the frozen `CONTRACT.md` (grep check against the file at `state.contract.path`). This run's own G2 already dispositioned cleanly (`decisions.md:37-38`: "gh CLI only, verified live; no unprocured rows") — the mechanism formalizes what this run already did by hand.

**D9 — per-epic Tier Table grammar (C-W3-01, C-W3-03).** New `references/formats.md` §10. Four required columns, exactly the vocabulary CONTRACT.md's RUN-POLICY block already uses (`CONTRACT.md:55-60`, verbatim keys `coordinator`/`mechanical`/`spec'd implementation`/`judgment`) plus the two G2-decided additions (`.plan-it/state.json:29-32`, Q3 = `scaffold-pointer-to-fable-it`):
```
| slice class | tier | escalation | scaffold-pointer |
|---|---|---|---|
| spec'd implementation vs Test Contract | mid | +1 tier on fail/flaky (RUN-POLICY) | fable-it:launch#slice=<eid> |
```
`scaffold-pointer` grammar: `fable-it:<skill-or-agent-id>[#<param>=<value>]` or a repo-relative agent-definition path (`.claude/agents/<name>.md`) — **no whitespace**, which is the exact discriminator C-W3-03 needs ("pointer must be a fable-it path, not inline prompt text"): prose always contains spaces, a path/pointer grammar never does.

**D10 — `planit-guard.mjs` hardcoded-model-ID rejection (C-W3-02).** Extends the existing hook per architecture C8 ("any new v3 PreToolUse guard extends planit-guard.mjs rather than adding a second hook," `v3-architecture.md:97-99`) — additive second check function alongside the existing Rule-1 deliverable gate (`scripts/hooks/planit-guard.mjs:21,40-65`). New `MODEL_ID_RE = /claude-[a-z0-9-]+/` (verbatim regex from `CONTRACT.md:23`) scanned against `tool_input.content ?? tool_input.new_string ?? ""` whenever the target path matches a broadened plan-artifact pattern (`delivery/**`, `docs/**`, `references/**`, `CONTRACT.md`, `SKILL.md`, `machine*.json`) — this check is **not** gated by `contract.version` (unlike Rule 1): a hardcoded model ID is always wrong, frozen or not. Denies with the offending literal named in the reason, same JSON protocol as the existing `deny()` helper (`planit-guard.mjs:27-38`).

**D11 — `freeze` verb checks the RUN-POLICY block (G1).** `gate-check.mjs:65-86` (`cmdFreeze`) currently only checks a generic `vN.N` header, ≥3 `##` sections, a changelog mention, and no placeholders — it has no opinion on section *content*. Add: `text` must contain a `## RUN-POLICY` heading, and that section's body must contain the literal strings `reap-on-merge` and both halves of the disk-AND-message delivery rule (`writes results to disk` … `content-bearing final message` — already present verbatim at `CONTRACT.md:63`). Additive: only a new failure branch inside `cmdFreeze`, existing checks unchanged.

**D12 — RUN-POLICY content-provenance (G2) + Tier-Table reconcile validators (C-W3-01/03, cross-squad).** `gate-check reconcile` is a new subcommand this run's contract requires (`C-W3-01`, `C-W5-02/03/04`) but whose *skeleton* (orphan-requirement detection, zero-case-epic detection, standalone-and-embedded-in-handoff wiring) is Squad A's W5 lane. This squad's contribution, landing as an additive function called from within that skeleton: (a) `checkRunPolicySeeded(contractText, stateJson)` — every value in `CONTRACT.md`'s RUN-POLICY table must trace to a verbatim key in `state.gates.G1.decisions` (`state.json:16-21` already has `tiering: "haiku fetch / sonnet analyze; ..."` — the check normalizes tier words, not literal sentence match); (b) `checkEpicTierTable(epicText)` — every epic section must contain a table with the D9 four fields, and any `scaffold-pointer` cell containing whitespace fails. Coordination flagged in Risks §6.

**D13 — cross-squad epic-ID grammar gap (PROPOSED-AMENDMENT-2).** `gate-check.mjs`'s existing `handoff` verb hardcodes a literal `E` in its test-ID and epic-heading regexes: `/\bT-E\d+[A-Za-z0-9.]*-\d{2}\b/g` (`gate-check.mjs:115`), `/^T-(E\d+)/` (`gate-check.mjs:121`), and `/^#{2,3}\s+(E\d+[^\n]*epic[^\n]*|epic\s+E\d+[^\n]*|E\d+\s+—[^\n]*)/gim` (`gate-check.mjs:161`) — written when v2's only epics were `E1..E6`. `00-program-plan.md:9` assigns this squad epic IDs `B1, B2, …` (Squad A/C presumably `A1../C1..`); as written, none of those will be recognized as valid `T-<EID>-NN` test IDs or epic headings by `handoff`/`reconcile` once assembled into the delivery package, even though `references/formats.md:120-127` already documents the grammar generically as `T-<EID>-NN`. **Recommendation:** widen both regexes' hardcoded `E` to `[A-Z]` — purely additive (every existing `T-E\d+-NN` ID still matches) — as a single coordinated patch, not three duplicate ones. Flagging to the coordinator to assign single ownership (suggest Squad A, which already owns `gate-check.mjs`'s W1/W5 lane) rather than each squad patching the same three lines independently. This squad's own epics below use `T-B<n>-NN` IDs on the assumption this widening lands; `T-B2-0x` below includes a verification task, not a duplicate implementation, pending coordinator assignment.

## 6. Risks

- **Shared-file collision.** `gate-check.mjs` and `scripts/hooks/planit-guard.mjs` each receive additive edits from this squad AND (independently) from Squad A/C on the same files (`commands` map at `gate-check.mjs:264-265`; the `deny()`/`allow()` protocol in `planit-guard.mjs`). No line ranges literally overlap in this PRD's design, but merge order across `epic/a*`, `epic/b*`, `epic/c*` branches needs a deliberate sequence, not three independent PRs landing blind. Surfaced to the coordinator alongside PROPOSED-AMENDMENT-2.
- **C-W2-03 cross-squad dependency.** This PRD defines the ABSENT-tool *schema*; Squad A's `contract` verb is the consumer. `T-B1-05` cannot go green until both land.
- **"contract" edge-target interpretation (D1).** If the coordinator's intended reading of C-W2-01's edge chain differs from D2's reachability interpretation, `T-B1-01` needs re-authoring, not a design rewrite — low blast radius, flagged explicitly rather than guessed silently.
- **Tier Table content-provenance check (D12b) is a soft parse.** Free-text tiering decisions in `state.json.gates.G1.decisions` (e.g. `"haiku fetch / sonnet analyze"`) are prose, not a structured list — `checkRunPolicySeeded` normalizes on tier words, which is inherently fuzzier than the other, purely-structural checks in this PRD. Acceptable given `C-W3-02`'s regex guard is the hard backstop against any *specific* model ID leaking through regardless.
